架構方案

如何將三個功能整合進現有專案？

A

漸進式整合（推薦）

保留現有元件結構，逐步加入新功能。改動最小，風險最低。

  • 路由：HashRouter 包住 App，列表頁 #/ + 詳情頁 #/project/:id
  • 搜尋：新增 SearchBar 元件，插在 FilterBar 上方，搜尋狀態提升到 App
  • 主題：ThemeToggle 元件 + CSS 變數切換（[data-theme="light"]），localStorage 記住偏好
  • 詳情頁：新增 ProjectDetail 元件，從 projects.json 取資料
  • 動畫：CSS transition 處理頁面切換（無需額外動畫庫）

Pros

  • 改動集中，現有元件幾乎不動
  • 不引入多餘依賴（只加 react-router-dom）
  • 可分步驟完成

Cons

  • CSS 變數需手動定義 light 版本
B

Context + 全域狀態重構

引入 React Context 統一管理 theme / search / filter 狀態，元件透過 hook 取值。

  • AppContext：統一管理 theme、searchQuery、activeCategory
  • useTheme / useSearch：自訂 hook 存取狀態
  • 其餘路由、元件同方案 A

Pros

  • 狀態管理更結構化
  • 未來擴充容易

Cons

  • 現階段只有 3 個狀態，Context 偏重
  • 改動範圍較大
